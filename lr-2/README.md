# Лабораторная работа 2. Шаблон «Декоратор»

Выполнил: Иванишин Даниил Евгеньевич

Программа для получения курсов валют от Центробанка РФ и их преобразования в форматы JSON, YAML и CSV с использованием структурного паттерна проектирования «Декоратор» (Decorator).

## Архитектура решения

Решение спроектировано по модульному принципу:

1. **interfaces.py**:
   - **`Component`** — абстрактный базовый класс, определяющий интерфейс для компонентов и декораторов. Метод `operation()` возвращает данные о курсах валют.
   - **`Decorator`** — базовый декоратор, который оборачивает любой другой `Component`. Содержит метод `save_to_file(filepath)` и предоставляет внутренний метод `_get_data()`, позволяющий вложенным декораторам прозрачно распаковывать форматированные строки обратно в словари.
2. **cbr_component.py**:
   - **`ConcreteComponent`** — конкретный компонент бизнес-логики. Загружает XML-данные от Центробанка РФ (`https://www.cbr.ru/scripts/XML_daily.asp`) с установленным тайм-аутом (`timeout=10`), парсит их с помощью `xml.etree.ElementTree` и возвращает структурированный словарь (`dict`).
3. **decorators.py**:
   - **`JsonDecorator`** — форматирует данные в JSON.
   - **`YamlDecorator`** — форматирует данные в YAML (требуется `PyYAML`).
   - **`CsvDecorator`** — форматирует данные в плоский CSV-список с заголовками.
4. **main.py**:
   - Точка входа для демонстрационного запуска.

## Запуск демонстрационного примера

Для работы с YAML требуется установить зависимость:

```bash
pip install -r requirements.txt
```

Для демонстрации получения реальных курсов валют и их сохранения в файлы запустите основной модуль:

```bash
python main.py
```

После выполнения скрипта в текущей директории будут созданы файлы `rates.json`, `rates.yaml` и `rates.csv`.

## Тесты

Для проверки корректности работы компонентов и декораторов написаны юнит-тесты на основе стандартной библиотеки `unittest`. Тесты покрывают:
- Успешный сетевой запрос и корректный парсинг XML от ЦБ (с использованием `unittest.mock`).
- Обработку сетевых ошибок при запросе.
- Форматирование данных каждым конкретным декоратором (JSON, YAML, CSV).
- Сохранение файлов каждым декоратором.
- Работу вложенных декораторов (`YamlDecorator` поверх `JsonDecorator`).

Запуск тестов:

```bash
python -m unittest test_currency.py -v
```
